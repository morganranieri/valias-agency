Context: 
  A user should, after their 30 day trial of the service, and after some grace period, be blocked from usage unless they pay.

Things that I need to figure out (
  Link off and back to the billing page
)

Views:
  In Settings (
    membershipScreen (contains the price, terms, opt-in or out)
    paymentsScreen (add, remove) methods
    wipeAccountScreen (wipe account)
  )

Model: 
  Date user.subscripton-due
  Date user.grace-until (to be clear, not globally inferred)
  Enum user.subscription-status (trialling, subscribed, grace, expired)

Controllers: 
  On user signup (
    set user.subscription-due to be due 30 days after;
    set user.grace-until to be user.subscription-due plus 7 days,
  )
  On user membership payment (
    upsert user.subscription-due to be what stripe returns as next payment due date (same day next month);
    upsert user.grace-until to be the 7 days after whatever stripe returns as next payment due date,
  )


View: 
  If (user.membership-status == in-grace) { 
    show gentleWarningModal (
      informing them that their membership will be deactivated on $user.grace-until with a link to settings/
      don't disappear too easily, consider repeating
      consider triggering a warning email
    )
  }

View: 
  If (user.grace-until is in the past) {
    show blockerModal (
      covers each view (days, dashboard, ins, outs) except settings
      with a link to settings/
    )
  }

Controller: 
  On payment fail (
    email the customer
    retry the payment two days later
    repeat until user.grace-until is past or user turns off their membership 
  )
